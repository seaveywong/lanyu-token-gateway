# 风险边界：网页登录账号池

用户希望支持 Plus/Pro 登录账号资源。该能力存在明显风险：

- 订阅账号通常不等于 API 转售授权。
- Web 登录接口不稳定，可能触发验证码、风控、封号。
- 自动化网页登录、Cookie 池、Session 共享、逆向 Web API 可能违反服务条款。
- 不能稳定统计 token、成本、上下文、工具调用和计费。
- 客户数据经过非官方 API 链路，隐私和审计风险高。

## 可接受路线

- 官方 API Key 池。
- 企业/团队项目 Key 池。
- 云厂商服务账号。
- 用户显式提供的官方 API Key。
- 官方 OAuth/授权流程。

## 不应实现的内容

- 模拟网页登录。
- 自动绕过验证码/二次验证。
- Cookie/Session 池。
- 逆向 ChatGPT/Claude/Gemini 网页接口。
- 将个人 Plus/Pro 订阅权益包装成对外 API 转售。

## 如果后续坚持调研

只能作为合规评估文档处理：记录平台条款、授权方式、封禁风险、数据安全影响，不写绕过型实现代码。
